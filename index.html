Can We Trust LLMs to Manage Time?

Motivating scenario: from conflict overload to confident, preference-aware scheduling with PEARL.

Introduction

Overlapping calendar invitations force busy professionals to repeatedly decide which meetings to attend, reschedule, or decline. We refer to this preference-driven decision process as calendar conflict resolution. Automating such process is crucial yet challenging: Scheduling logistics drain hours, and human delegation often fails at scale, raising the question of whether large language model (LLM) or language agents can reliably learn and apply user preferences to manage time.

To enable systematic study, we introduce CALCONFLICTBENCH, a benchmark for long-horizon calendar conflict resolution. Conflicts are presented sequentially and agents receive feedback after each round, requiring them to infer and adapt to user preferences progressively. Our experiments show that current LLM agents perform poorly with high error rates. To address this gap, we propose PEARL, a reinforcement-learning framework that augments language agent with an external memory module and optimized round-wise reward design, enabling agent to progressively infer and adapt to user preferences on-the-fly. Experiments on CALCONFLICTBENCH shows that PEARL achieves 0.76 error reduction rate, and 55% improvement on average error rate compared to the strongest baseline.

CalConflictBench: Can we trust LLMs to manage time?

Scheduling logistics drain hours each week. While LLMs show promise, it remains unclear if they can handle the nuances of personal time management where preferences drift and constraints are complex.

We propose the CALCONFLICTBENCH, a new benchmark featuring:

  • Synthetic Users: Diverse organizational roles and year-long calendars.
  • Long-Horizon: Conflicts presented sequentially; agents must learn from feedback.
  • Complex Conflicts: Scenarios requiring balancing urgency, hierarchy, and personal values.

PEARL: Preference Evolving Agent with RL

Overview of PEARL. The agent uses an external Strategy Hub to store and update interpretable decision strategies, optimized via a curriculum-based reward model.

Strategy Hub

An external memory module that stores interpretable decision strategies (preference states). It is iteratively retrieved and updated to capture newly revealed user priorities.

Curriculum Rewards

Optimized round-wise reward design that shifts emphasis from preference inference in early rounds to consistent decision-making in later rounds.

Self-Evolving

The agent adapts on-the-fly. Unlike static prompts, PEARL refines its policy based on interaction history to align with shifting user constraints.
